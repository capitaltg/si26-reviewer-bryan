# Extraction Evidence-Quote Grounding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace extraction's brittle verbatim-`text` page check with a required `evidence_quote` span whose match result is persisted without aborting extraction when the span is non-matching or too short.

**Architecture:** `text` stays the model's paraphrase (consumed downstream unchanged). A new required `evidence_quote` field is validated against the cited page via the existing ZWSP-aware `_normalize_quote`; the pass/fail result is persisted as `grounding_verified` alongside the stored quote. Bad grounding degrades to `grounding_verified = false` (fail-open), while structural citation errors (bad doc/page) still raise.

**Tech Stack:** Python 3.12+ worker (`psycopg`, `pydantic`, `anthropic`/Bedrock), Postgres, Drizzle ORM migrations (`web/src/db/schema.ts` → `web/drizzle/*.sql`), pytest.

---

## Global Constraints

- Worker + database only. No web UI changes; no `verify.py`/mapping/reviewer/orchestrator changes.
- `evidence_quote` is a **required** model/tool field, validated non-blank via the existing `_trimmed` validator; missing/blank → `ExtractionError`.
- `grounding_verified = true` iff `len(_normalize_quote(evidence_quote)) >= 20` **and** `_normalize_quote(evidence_quote)` is a substring of `_normalize_quote(page_text)` for the cited page. The 20 counts normalized Python string characters.
- The flag proves quote occurrence only, not semantic entailment of the
  paraphrased `text`. Requirements with `grounding_verified = false` continue
  downstream unchanged in this fail-open scope.
- A failed grounding check **never raises** — set `grounding_verified = false` and persist the requirement anyway.
- Existing hard validation errors stay `ExtractionError`s: out-of-range `[doc N]` handle, nonexistent page, duplicate keys, supersession cycle/self-reference.
- Migration columns: `evidence_quote text NOT NULL DEFAULT ''`, `grounding_verified boolean NOT NULL DEFAULT false`. Existing rows keep the defaults (unverified).
- `_validate_result` returns `tuple[dict[str, str | None], dict[str, bool]]` = `(supersedes_by_key, grounding_by_key)`.
- Worker tests run from `worker/` as:
  `TEST_DATABASE_URL="postgres://postgres:dev@localhost:5435/agentic_review_test" .venv/bin/python -m pytest <args>`
  The test DB is rebuilt from `web/drizzle/*.sql` by `conftest.py` on every test, so a generated migration is picked up automatically.

**Execution prerequisite:** Use a clean dedicated worktree based on a commit that
already contains the two prerequisite fixes named by the design: Unicode format

`extra="ignore"` on all three extraction Pydantic models. Before Task 1, run:
